# 아껴맵 요구사항

## 서비스 목적

아껴맵은 대학생과 취업준비생이 학교 주변의 저렴한 식당, 무료 학습 공간, 생활 편의시설을 지도 기반으로 찾고 공유할 수 있는 생활비 절약 지도 서비스입니다.

핵심은 사용자가 직접 제보한 장소 정보를 바로 노출하지 않고, 관리자 승인과 신고 처리를 통해 신뢰도 있게 운영하는 것입니다.

## 주요 사용자

### 일반 사용자

- 학교 주변의 저렴한 장소를 지도에서 찾습니다.
- 새로운 장소를 제보합니다.
- 승인된 장소의 상세 정보, 가격 정보, 리뷰를 확인합니다.
- 잘못된 장소 정보나 부적절한 내용을 신고합니다.

### 관리자

- 사용자가 등록한 장소를 검토합니다.
- 장소를 승인, 반려, 숨김, 복구 처리합니다.
- 신고된 장소를 확인하고 운영 상태를 관리합니다.

## MVP 기능

### 1차 구현 범위

- 장소 등록
- 등록된 장소의 `PENDING` 상태 저장
- 관리자 장소 승인
- 승인된 장소만 목록 조회
- 장소 상세 조회
- 지도 마커 표시용 장소 조회

### 2차 구현 범위

- 관리자 장소 반려
- 관리자 장소 숨김 및 복구
- 리뷰 작성
- 장소 신고

### 이후 확장 범위

- 로그인
- JWT 인증
- `USER` / `ADMIN` 권한 분리
- 즐겨찾기
- 내가 등록한 장소 조회
- 가격 변경 제보
- 신고 누적 자동 숨김
- 거리순 정렬
- 추천순 정렬

## 사용자 시나리오

### 장소 등록

1. 사용자가 장소 이름, 카테고리, 주소, 좌표, 가격 정보, 설명을 입력합니다.
2. 서버는 장소를 `PENDING` 상태로 저장합니다.
3. 등록된 장소는 관리자 승인 전까지 지도와 일반 목록에 노출되지 않습니다.

### 승인된 장소 조회

1. 사용자가 장소 목록 또는 지도를 엽니다.
2. 서버는 `APPROVED` 상태의 장소만 반환합니다.
3. 사용자는 장소 상세 정보와 가격 정보를 확인합니다.

### 장소 신고

1. 사용자가 잘못된 가격, 폐업, 중복 장소 등 문제를 발견합니다.
2. 사용자는 신고 사유와 내용을 제출합니다.
3. 서버는 신고 내역을 저장합니다.
4. 관리자는 신고 내역을 검토한 뒤 숨김 또는 복구 처리를 합니다.

## 관리자 시나리오

### 장소 승인

1. 관리자가 승인 대기 장소 목록을 조회합니다.
2. 장소 정보가 적절하면 승인합니다.
3. 장소 상태가 `APPROVED`로 변경됩니다.
4. 승인된 장소는 일반 목록과 지도 마커 API에 포함됩니다.

### 장소 반려

1. 관리자가 승인 대기 장소를 검토합니다.
2. 정보가 부정확하거나 부적절하면 반려합니다.
3. 장소 상태가 `REJECTED`로 변경됩니다.

### 장소 숨김 및 복구

1. 관리자가 신고가 접수된 장소를 검토합니다.
2. 노출하면 안 되는 장소는 `HIDDEN` 상태로 변경합니다.
3. 문제가 해결된 장소는 `APPROVED` 상태로 복구합니다.

## MVP 완료 기준

- 장소 등록 API가 동작합니다.
- 등록된 장소는 기본적으로 `PENDING` 상태로 저장됩니다.
- 관리자가 장소를 승인할 수 있습니다.
- 승인된 장소만 일반 목록에 조회됩니다.
- 승인된 장소만 지도 마커 API에 포함됩니다.
- 장소 상세 조회가 가능합니다.
- 사용자는 장소를 신고할 수 있습니다.
- 관리자는 신고된 장소를 숨김 처리할 수 있습니다.
- 주요 Service 로직에 테스트 코드가 존재합니다.
- README에 프로젝트 소개, 실행 방법, 주요 기능, API 요약이 정리되어 있습니다.
